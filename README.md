# NFL 4th Down Decision Analysis

**Are NFL coaches making optimal 4th down decisions — and is Dan Campbell actually a genius?**

A data-driven analysis of every NFL 4th down decision from 1999–2024 using nflfastR play-by-play data.

## Methodology

### 1. Data
- **Source:** nflfastR via nflverse GitHub (free, no API key required)
- **Scope:** 1999–2024 regular seasons, 4th down plays only
- **Decisions classified:** Go for it / Punt / Field goal attempt

### 2. Optimal Decision Framework
For each game state (field position × yards to go × score differential × time remaining), we compute the **historically weighted average WPA** (Win Probability Added) for each available decision. The decision with the highest WPA in that game state is labeled **optimal**.

Recent seasons are weighted more heavily using exponential decay (decay=0.85 per year) to account for rule changes and evolving coaching philosophy.

### 3. Coach Grading
Two metrics per coach:

| Metric | Definition | Direction |
|--------|-----------|-----------|
| **DQS** (Decision Quality Score) | Mean gap between optimal WPA and actual decision WPA | Lower = better |
| **ODR** (Optimal Decision Rate) | % of plays where coach made the optimal call | Higher = better |

Minimum 50 decisions required to qualify.

### 4. Dan Campbell Deep Dive
Situational breakdown of Campbell's decisions by:
- Field position
- Yards to go
- Season-over-season trend
- Best and worst individual calls

## Setup

Use the same Python you select as the Jupyter kernel (Anaconda: activate your env first). On Windows, prefer `python -m pip` so installs go to that interpreter.

**EDA and data pipeline** (`01_eda.ipynb`):

```bash
python -m pip install -r requirements.txt
```

**Optional — modeling** (later notebooks that use XGBoost / Optuna / SHAP):

```bash
python -m pip install -r requirements-ml.txt
```

If a one-liner fails, install the core file first; ML wheels are heavier and sometimes need a newer `pip` (`python -m pip install --upgrade pip`).

Then run notebooks in order:
```bash
jupyter notebook notebooks/01_eda.ipynb
```

Data downloads automatically on first run and caches locally — no credentials needed.

## Limitations
- WPA baselines are league averages and don't account for roster quality (e.g., having Patrick Mahomes changes the math)
- Small sample sizes in rare game states (very late, extreme score differentials) make those baselines less reliable
- Coach assignments are approximate — offensive coordinators make most play calls but head coaches set the aggressive/conservative philosophy
- This analysis identifies correlation, not causation — a coach having a poor DQS might reflect roster constraints rather than bad decisions
